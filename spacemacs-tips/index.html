Spacemacs

“I’m using Linux. A library that Emacs uses to communicate with Intel hardware.” – Erwin, #emacs, Freenode.

Introduction and motivation

As I prefer to work on other things than studying documentation for how to use certain tools, I will keep this introduction very short.

In short Emacs is a universal text editing environment that is implemented as and powered by a virtual lisp-machine. Spacemacs is a community based set of configurations which focuses on vim-style editing with the notable - and in my opinion genius - improvement of using the space bar as a leader key for navigation and controlling the lisp-machine. A crucial difference between Emacs and any other text editor is its focus on extensibility and self-documentation which makes it the ultimate editor for hackers and community driven development.

Spacemacs tips

Use the Spacemacs develop branch, it is really recommended by most developers (e.g. on Spacemacs gitter)

Follow (ALL) the steps at the great practicalli website to switch to develop branch.

Use o instead of w for editing of symbol(-name)

In spacemacs a word(-object) is defined differently than in vim. To quickly edit full symbol name use o instead of w (e.g. for selecting a symbol use v i o).

Quick navigation within files

To quickly navigate inside a file use SPC j i. Additionally one could choose to set a local variable to start with the code folded using HideShow. E.g. replace the first line of your .spacemacs file by the following: 

;; -*- mode: emacs-lisp; eval: (hs-hide-all); lexical-binding: t -*-

Comments

Lines can be (un)commented by selecting line with evil visual line SPC V followed by SPC ;. Use SPC ; ; to directly toggle single line comments.

Finding/discovering functionality

When looking for some kind of functionality or it's keybinding, it is often handy to use the self-documenting features of Spacemacs.

For example to find how to paste (which is called yank in emacs) in evil-insert mode, type M-m h d b while in insert mode, and type "yank" (including double quotes) to quickly find keybindings to the yank function.

As shown in the link above, there are many more describe functions (see SPC h d). Note them well, THEY ARE PROBABLY THE MOST IMPORTANT FUNCTIONS ONE SHOULD KNOW FOR MASTERING (SPAC)EMACS

Private packages/layers

Instead of creating private layers directly in your .emacs.d/private/ directory, consider to place them in a dedicated syncing directory (e.g. a folder in your Syncthing or Dropbox directory) and then create symbolic links in your .emacs.d/private/ directory. Even better create a github repository per layer.

Org notes

Create an org folder in some syncing directory (e.g. Dropbox or Syncthing). Subsequently in that org folder create a .projectile file. Now you can use all projectile functions e.g. SPC p f to open a file within the project, or SPC / to search withing all your notes files.

Deep-search

Remember that ripgrep, ag etc. ignore files in .gitignore. Therefore you can not simply use them to search e.g. inside of elpa packages in your .emacs.d directory. Instead either use SPC s f or use grep SPC s g there.

Customizations

  • set dotspacemacs-distinguish-gui-tab to t in your .spacemacs file so you can jump forward with C-i after jumping backward with C-o.

Place (transient-bind-q-to-quit) at the end of user-config

I think this should be the default configuration in Spacemacs, because it is more vim-ish (you can find a discussion here).

Annotate PDF's

To make annotating pdf's more comfortable, set pdf-annot-activate-annotation to t and add text-mode to evil-insert-state-modes

Customize buffer newline

In customize buffer, for capture template, to create a newline (instead of \n charachter in config file), use C-j or C-o reference link

Spell check language

To get Dutch spell check install Dutch aspell dictionary and restart emacs (or maybe SPC f e R will work too)

help/introspection

This is the most important table. You should note remember and remember to use all options. Therefore study this list carefully.

IMPORTANT COMMENTS:

  • In info mode most of the evil-bindings are not well

implemented. Therefore it might be handy to read the Emacs documentation about how to navigate the info manuals.

  • The describe function is extremely handy. You should study all describe options, (especially: bindings, function, key, mode, variable).

Although the describe function and variable functionalities might get useful only when writing or studying lisp, the other options are really handy for quick help lookups.

  • With the binding option you can quickly lookup key-bindings by fuzzy search on function name (e.g. 'upcase', 'comment', 'jump', 'delete').
  • With the key option you can quickly find the function name bound to a key-sequence by searching on key sequence (e.g. SPC l, or g g).
  • With the mode option you can quickly get information about the current major mode (e.g. special functions and bindings for the python mode)

help/introspection

Go to info C-h i
Describe ANYTHING SPC h d
Execute (or choose action on) function SPC SPC

SPC SPC opens HELM M-x, you can type a name of any function and either execute it or type C-z to choose from a list of alternative actions (e.g. describe, go to definition)

When the cursor (usually called POINT in Emacs) is on a function or variable name, one can use their respective describe key-sequences to quickly get info about them.
